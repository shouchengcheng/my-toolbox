# 本地日历待办工具 使用说明文档

## 一、工具简介
这是一款纯前端单文件待办管理工具，基于 HTML + JavaScript 开发，**无需安装任何运行环境、无需联网**，双击文件即可运行。工具以月历为核心视图，支持待办创建、进度追踪、到期提醒、数据备份等完整功能，所有数据完全本地存储，无任何网络上传，保护隐私安全。

## 二、完整功能列表
### 1. 月历视图
- 按月份展示日历网格，直观查看每日待办分布
- 支持「上一月 / 下一月」快速切换，「今天」一键返回当日
- 支持年月下拉框快速跳转到任意指定年份月份
- 当日日期蓝色高亮标记，非当月日期灰色弱化显示
- 每个日期格子最多展示 3 条待办，超出自动显示「+N 更多」提示

### 2. 待办全生命周期管理
- **添加待办**：点击任意日期格子即可创建，支持设置标题、详细内容、提醒时间
- **查看详情**：点击待办条目打开当日待办详情，查看完整内容、创建时间、提醒时间、完成状态
- **编辑待办**：支持修改标题、内容、提醒时间；清空提醒时间输入框即可取消提醒
- **删除待办**：详情页和编辑页均支持删除操作，删除前弹出二次确认
- **状态切换**：一键勾选切换待办完成/未完成状态，自动记录完成时间
- **视觉区分**：已完成待办显示绿色删除线样式，未完成待办为蓝色，状态一目了然

### 3. 连续进度记录
- 详情页和编辑页均内置「快速追加」输入框
- 输入进度内容后回车或点击按钮，**自动追加带精确时间戳的记录**
- 自动换行排版，非常适合连续记录任务中间状态、问题排查过程、工作日志
- 同时保留完整文本编辑框，支持大段内容的整理与修改

### 4. 系统通知提醒
- 基于浏览器原生 Notification API 实现系统级桌面通知
- 每条待办可单独设置精确到分钟的提醒时间
- 到点自动弹出系统通知，点击通知可直接跳转到对应日期月历
- 同一条待办仅提醒一次，修改提醒时间后自动重置提醒状态
- 待办标记为完成后，对应提醒自动失效

### 5. 全量待办汇总
- 底部汇总表展示所有待办，默认按创建时间倒序排列
- 支持状态筛选：全部 / 未完成 / 已完成
- 分页展示（每页 50 条），数据量再大也不会卡顿
- 列表字段：所属日期、待办标题、创建时间、状态、完成时间
- 设置了提醒的待办会显示橙色提醒标识和具体提醒时间

### 6. 快捷跳转
- 汇总表「所属日期」列点击即可跳转到对应月份的月历视图，并平滑滚动到日历顶部
- 点击待办标题直接打开编辑弹窗，快速修改内容

### 7. 数据备份与迁移
- **导出数据**：一键导出所有待办为独立的 JSON 备份文件
- **导入数据**：选择备份文件即可完整恢复全部数据，支持跨设备、跨浏览器迁移
- 导入前弹出二次确认，防止误操作覆盖现有数据

### 8. 性能优化
- 内存缓存机制：页面加载时仅读取一次本地数据，后续所有操作直接读写内存
- 防抖持久化：数据修改后延迟写入本地存储，避免频繁 IO 阻塞主线程
- 分页渲染：汇总表仅渲染当前页 DOM，性能不随总数据量增长而下降
- 文档碎片优化：批量渲染 DOM 时减少重排重绘，提升页面流畅度

## 三、使用方法
### 1. 运行方式
将代码保存为 `calendar-todo.html` 文件，使用任意现代浏览器（Chrome、Edge、Firefox、Safari 等）双击打开即可使用，无需其他任何操作。

### 2. 基础操作
1. **添加待办**：点击日历中任意日期格子，在弹窗中填写标题和内容，点击「添加」即可
2. **查看详情**：点击日期格子内的待办文字，打开当日所有待办的详情弹窗
3. **标记完成**：在详情页勾选待办前的复选框，切换完成状态
4. **追加进度**：在详情页/编辑页的「快速追加」输入框输入内容，回车即可追加带时间戳的记录
5. **编辑/删除**：点击详情页「完整编辑」按钮，或直接点击汇总表的待办标题，进入编辑页修改或删除

### 3. 开启提醒
1. 点击汇总表上方的「开启提醒」按钮，在浏览器弹出的权限申请中选择「允许」
2. 添加或编辑待办时，在「提醒时间」栏选择具体的日期和时间
3. 保持页面打开，到点会自动弹出系统通知

### 4. 数据备份
1. **导出备份**：点击汇总表上方「导出数据」，自动下载 JSON 备份文件，建议定期备份
2. **恢复数据**：点击「导入数据」，选择之前导出的 JSON 文件，确认后即可覆盖恢复所有数据

## 四、数据存储说明
本工具采用「**本地缓存 + 手动备份**」双层存储机制：

### 1. 运行时存储：浏览器 localStorage
- 日常使用时，所有待办数据自动保存在浏览器的 `localStorage` 中
- 存储位置与当前网页文件路径、浏览器一一绑定，不同浏览器、不同文件路径下数据不互通
- 数据随浏览器本地缓存持久化保存，关闭页面再打开数据依然保留
- 存储容量上限约 5MB，纯文本待办可存储数万条，完全满足日常使用需求

### 2. 独立备份文件：JSON 格式
- 通过「导出数据」功能生成的 `.json` 文件是完整的数据副本
- 文件可保存到电脑任意位置、U盘、网盘等，作为长期备份
- 数据为明文结构化格式，可手动查看、编辑，不绑定工具本身
- 换设备、换浏览器时，通过导入该文件即可完整迁移所有数据

### 3. 数据字段说明
每条待办包含以下字段：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | 数字 | 待办唯一标识，由时间戳生成 |
| title | 字符串 | 待办标题 |
| content | 字符串 | 详细内容与进度记录 |
| remindTime | 数字/null | 提醒时间戳，null 表示无提醒 |
| reminded | 布尔值 | 是否已提醒过 |
| createTime | 数字 | 创建时间戳 |
| done | 布尔值 | 完成状态 |
| finishTime | 数字/null | 完成时间戳 |

数据按日期分组存储，结构示例：
```json
{
  "2026-06-15": [
    {
      "id": 1718409600000,
      "title": "调试WiFi模组",
      "content": "[2026-06-15 09:30:12] 排查断连问题",
      "remindTime": 1718438400000,
      "reminded": false,
      "createTime": 1718409600000,
      "done": false,
      "finishTime": null
    }
  ]
}
```

## 五、注意事项
### 1. 数据安全
- **请勿随意清理浏览器缓存/站点数据**：清理浏览器数据会导致 localStorage 中的待办数据永久丢失
- **定期导出备份**：建议每周或每月导出一次 JSON 备份，防止误操作、浏览器故障、系统重装导致数据丢失
- 删除待办操作不可恢复，重要数据删除前请仔细确认

### 2. 提醒功能
- 提醒功能**仅在页面保持打开时生效**，关闭浏览器或页面后无法推送提醒
- 首次使用需授权浏览器通知权限，拒绝权限后无法使用提醒功能
- 系统勿扰模式、专注模式会拦截系统通知，请确保系统通知权限正常开启
- 提醒每分钟轮询检测一次，可能存在最多 1 分钟的延迟，适合非高精度场景

### 3. 兼容性与环境
- 推荐使用 Chrome、Edge、Firefox 等现代浏览器，IE 浏览器不支持
- 不同浏览器之间数据不互通，跨浏览器使用请通过导出导入功能迁移
- 纯本地运行，无需联网，无任何数据上传，完全隐私安全

### 4. 文件使用
- 单 HTML 文件可任意移动、重命名，不影响功能本身
- 若将文件移动到其他路径或重命名，浏览器会视为新站点，原有 localStorage 数据不会自动同步，需导入备份文件
- 建议固定文件存放位置，避免频繁移动导致数据分散

### 5. 容量与性能
- localStorage 约 5MB 容量限制，纯文本待办可存储数万条，正常使用无需担心容量
- 得益于分页渲染和内存缓存优化，即使上万条待办，操作响应依然流畅
- 数据量极大时，导出的备份文件体积依然很小，便于传输和存储

## 六、常见问题
**Q：浏览器清理缓存后数据没了怎么办？**
A：如果之前导出过备份文件，通过「导入数据」功能恢复即可；未备份则无法恢复，建议定期备份。

**Q：到点没有收到提醒？**
A：请依次检查：① 页面是否保持打开状态；② 浏览器通知权限是否开启；③ 系统勿扰模式是否关闭；④ 提醒时间是否设置正确。

**Q：可以在手机上使用吗？**
A：可以，手机浏览器打开 HTML 文件即可使用，但提醒功能在移动端后台时可能失效。

**Q：数据会上传到网络吗？**
A：不会，所有数据均保存在本地浏览器和你导出的文件中，工具无任何网络请求，完全离线运行。
